Explore the thesis

"Why is there no operating system for the people who actually deliver products and services at the last mile?"

The founding insight

A legibility problem, not a motivation problem

Every company that reaches rural or underserved markets depends on a frontline agent — a community health worker, an agrodealer, a sales rep, a delivery rider. The infrastructure supporting them hasn't changed in 20 years: paper, WhatsApp, Excel, and a supervisor who can't see what's actually happening.

The problem isn't the agents. It's that every function touching them — onboarding, supervision, incentives, coaching, advisory — runs in its own silo, with its own data, with its own blind spots.

01

Frontline agents underperform because the systems managing them are structurally fragmented — not because they lack motivation or training.

02

When agent performance becomes governable — connecting functions, surfacing data — managers diagnose instead of guess. That's the studio's core thesis.

The studio model — why this, not a startup

Systematic exploration, not single inspired bets

Breakthroughs emerge from running parallel experiments across verticals — health, agri, FMCG — sharing the same infrastructure, the same behavioral design playbook, and the same data compounding from every deployment.

AdvantageShared infrastructure
Every venture built on the agent OS feeds performance data back to the platform. A deployment for CHWs in Kenya generates behavioral insights that improve the seed-dealer deployment in India. This compounding is structurally impossible for a standalone startup.
DisciplineProto-company approach
Each venture starts as a numbered experiment, not a named company. The first six months run the "killer experiment" — the test most likely to prove the thesis wrong. Only surviving configurations earn capital and a name.
RevenueArchitecture
Corporate co-builds and managed service contracts bridge the cash gap. Input companies, health ministries, and multinationals pay for execution quality, not technology. The studio earns fees while building equity in platform companies.

The investment arc

Three phases to platform dominance

1

Evidence — now

Deep integration with one anchor (e.g. Apollo Agriculture). Prove the thesis: connecting fragmented agent-management functions produces measurable performance lift. Generate the data and the reference architecture.

2

Platform — 18 months

Ship AgentOS as a product. Launch 2–3 proto-companies in health and agriculture. Run killer experiments on each value proposition. Fund the ones that do.

3

Scale

Surviving configurations enter new markets and sectors. Multinationals anchor Wave 2 ventures with co-build fees. The platform's compounding data advantage becomes a defensible moat.

The structural moat

Behavioral data, not software

The studio's durable advantage is data — data on what actually works, what interventions work, and which incentive structures produce genuine performance improvement. Every vertical deployment makes the platform smarter for the next one. This is the same compounding logic that makes Flagship Pioneering defensible: not any single company, but the institutional capability to generate better companies than anyone else can.
